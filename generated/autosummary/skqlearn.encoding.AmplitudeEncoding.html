AmplitudeEncoding¶

class skqlearn.encoding.AmplitudeEncoding¶

Bases: skqlearn.encoding.encoding.Encoding

Amplitude encoding method. [SP18]

In amplitude encoding, each component of the input vector \(\boldsymbol{x} \in \mathbb{R}^N\) is mapped to an amplitude of the quantum state, defining the encoding feature map as:

\[\phi:\boldsymbol{x}\rightarrow\ket{\psi_\boldsymbol{x}}=\sum_{i=1}^{N} x_i\ket{i}\]

In order to represent a valid quantum state the amount of amplitudes, and therefore, the dimension of the vectors must be a power of 2, \(N=2^n\). If they are not, they will be padded with zeros at the end.

Therefore, the kernel defined by the inner product is the linear kernel:

\[k(\boldsymbol{x}, \boldsymbol{x'}) = \braket{\psi_{\boldsymbol{x}}| \psi_{\boldsymbol{x'}}} = \boldsymbol{x}^T\boldsymbol{x'}\]

A dataset can be encoded by concatenating all the input vectors.

encoding(x: numpy.ndarray) → numpy.ndarray¶

Application of amplitude encoding to the input.

Parameters

x (np.ndarray of shape (n_features,) or (n_samples, n_features)) –

Input. This can be a single sample of shape (n_features,) or a dataset of shape (n_samples, n_features).

Note

The input must be normalized in order to define a valid quantum state. Refer to ExpandedAmplitudeEncoding if the data is not normalized.

Returns

Quantum state described as an amplitude vector. If a dataset is provided, the states are concatenated.

Return type

np.ndarray

Raises

ValueError – If an invalid input type is provided or it is not normalized.
