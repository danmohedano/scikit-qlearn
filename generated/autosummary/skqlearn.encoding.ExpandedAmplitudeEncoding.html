ExpandedAmplitudeEncoding¶

class skqlearn.encoding.ExpandedAmplitudeEncoding(degree: int = 1, c: float = 1.0)¶

Bases: skqlearn.encoding.base_encoding.Encoding

Expanded amplitude encoding method. [SP18]

This encoding method tries to solve the normalization problem in regular Amplitude Encoding. If non-normalized data is normalized for use on Amplitude Encoding, the data will lose one dimension of information. For example, if a 2D point is normalized, it will be mapped into the unit circle, a 1D shape. By adding an extra component to \(\boldsymbol{x}\in\mathbb{R}^N\) with a value of \(c\), \(x_{0}=c\), and then normalizing, the information loss is mitigated.

The encoding and produced kernel work identical to regular Amplitude Encoding’s.

\[\phi:\boldsymbol{x}\rightarrow\left|\psi_\boldsymbol{x}\right>= \frac{1}{\sqrt{|\boldsymbol{x}|^2+c^2}}\left(c\left|0\right> + \sum_{i=1}^{N}x_i\left|i\right>\right)\]
\[k(\boldsymbol{x}, \boldsymbol{x'}) = \left<\psi_{\boldsymbol{x}}| \psi_{\boldsymbol{x'}}\right> = \frac{1}{\sqrt{|\boldsymbol{x}|^2+c^2} \sqrt{|\boldsymbol{x'}|^2+c^2}}(\boldsymbol{x}^T\boldsymbol{x'} + c^2)\]

As in Amplitude Encoding, the vectors can be mapped into copies of the state, defining the kernel:

\[k(\boldsymbol{x}, \boldsymbol{x'}) = \left<\psi_{\boldsymbol{x}}| \psi_{\boldsymbol{x'}}\right> = \left(\frac{1}{\sqrt{|\boldsymbol{x}|^2 +c^2}\sqrt{|\boldsymbol{x'}|^2+c^2}}(\boldsymbol{x}^T\boldsymbol{x'} +c^2)\right)^d\]

The kernel is then corrected by a factor of \(\sqrt{|\boldsymbol{x}|^2+c^2}\sqrt{|\boldsymbol{x'}|^2+c^2}\) in order to define a generic polynomial kernel.

Note

The normalization of the expanded vectors and application of the correction produce a very small rounding error in the values obtained for the kernel.

degree¶

Desired degree of the polynomial kernel defined by the encoding. In turn, it defines the amount of copies of each input vector that are encoded into the quantum state.

Type

int

c¶

encoding(x: numpy.ndarray) → numpy.ndarray¶

Application of expanded amplitude encoding to the input.

Expanded amplitude encoding allows for non-normalized vector to be encoded as quantum states. This is achieved by the inclusion of an extra feature/component with value \(c\).

Parameters

x (numpy.ndarray of shape (n_features) or (n_samples, n_features)) – Input. This can be a single sample of shape (n_features,) or a dataset of shape (n_samples, n_features).

Returns

Quantum state described as an amplitude vector. If a dataset is provided, the states are concatenated.

Return type

numpy.ndarray

Raises

ValueError – If an invalid input is provided.

Examples

>>> a = np.array([1.0, 1.0, 1.0])
>>> ExpandedAmplitudeEncoding(c=1.0).encoding(a)
array([0.5, 0.5, 0.5, 0.5])
