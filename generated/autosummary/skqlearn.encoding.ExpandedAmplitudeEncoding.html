ExpandedAmplitudeEncoding¶

class skqlearn.encoding.ExpandedAmplitudeEncoding¶

Bases: skqlearn.encoding.base_encoding.Encoding

Expanded amplitude encoding method. [SP18]

This encoding method tries to solve the normalization problem in regular Amplitude Encoding. If non-normalized data is normalized for use on Amplitude Encoding the data will lose one dimension of information. For example, if a 2D point is normalized, it will be mapped into the unit circle, a 1D shape. By adding an extra component to \(\boldsymbol{x}\in\mathbb{R}^N\) with a value of \(1\), \(x_{N+1}=1\), and then normalizing, the information loss is avoided.

The encoding and produced kernel are identical to regular Amplitude Encoding’s.

encoding(x: numpy.ndarray) → numpy.ndarray¶

Application of expanded amplitude encoding to the input.

Expanded amplitude encoding allows for non-normalized vector to be encoded as quantum states. This is achieved by the inclusion of an extra feature/component with value 1.

Parameters

x (np.ndarray of shape (n_features,) or (n_samples, n_features)) – Input. This can be a single sample of shape (n_features,) or a dataset of shape (n_samples, n_features).

Returns

Quantum state described as an amplitude vector. If a dataset is provided, the states are concatenated.

Return type

np.ndarray

Raises

ValueError – If an invalid input is provided.

Examples

>>> a = np.array([1.0, 1.0, 1.0])
>>> ExpandedAmplitudeEncoding().encoding(a)
array([0.5, 0.5, 0.5, 0.5])
