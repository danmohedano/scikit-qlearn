JobHandler¶

class skqlearn.utils.JobHandler¶

Bases: object

Singleton class to contain the configuration for quantum executions.

All quantum subroutines implemented in the package execute the quantum circuits through this handler. Therefore, before executing any of them, a backend should be correctly configured.

backend¶

Backend where the quantum circuits will be run.

Type

qiskit.proviers.Backend

shots¶

Amount of shots (repetitions) in the execution of the circuits.

Type

int

run_options¶

Dictionary to provide the backend with extra options when executing the run method.

Type

dict

compiled_circuits¶

Compiled circuits in the last run_job batch.

Type

Union[QuantumCircuit, List[QuantumCircuit]]

Examples

The JobHandler can be configured with local simulators.

>>> from qiskit.providers.aer import AerSimulator
>>> JobHandler().configure(AerSimulator(), 10000)

And with remote systems and simulators accessed through your IBMQ account. The following code will raise an exception because an invalid API token is provided. But describes the correct steps to follow.

>>> from qiskit import IBMQ
>>> try:
...     provider = IBMQ.enable_account('MY_API_TOKEN')
... except qiskit.providers.ibmq.api.exceptions.RequestsApiError:
...     pass
>>> try:
...     backend = provider.get_backend('ibmq_qasm_simulator')
... except NameError:
...     backend = None
>>> JobHandler().configure(backend, 10000)

configure(backend: qiskit.providers.backend.Backend, shots: int, run_options: dict = {})¶

Configuration of the job handler.

Parameters

  • backend (Backend) – Desired backend to use when running circuits (either simulator or real backends).

  • shots (int) – Number of executions.

  • run_options (dict) – Keyword dictionary used in the run method as run time backend options.

Raises

ValueError – If a non-positive amount of shots is provided.
